## D3. 接口契约模板（逐接口复制）

### 接口模板：获取设备主数据（基础数据平台集成）

- **接口名称**：获取设备主数据
- **URL/Topic**：`GET /api/v1/data-platform/equipments`
- **鉴权方式**：Bearer Token（平台统一认证） + 数据权限过滤
- **请求字段**：
  
  | 字段名 | 类型 | 必填 | 说明 | 示例 |
  |--------|------|------|------|------|
  | equipment_codes | string[] | 否 | 设备编码列表，多个用逗号分隔 | ["CNC001","ROBOT002"] |
  | category | string | 否 | 设备分类 | "CNC_MACHINE" |
  | status | string | 否 | 设备状态 | "ACTIVE" |
  | page | integer | 否 | 页码，从1开始 | 1 |
  | page_size | integer | 否 | 每页数量，默认20，最大100 | 20 |

- **响应字段**：
  
  ```json
  {
    "code": 200,
    "message": "success",
    "data": {
      "total": 150,
      "page": 1,
      "page_size": 20,
      "items": [
        {
          "equipment_code": "CNC001",
          "equipment_name": "五轴CNC加工中心",
          "category": "CNC_MACHINE",
          "model": "DMU-80",
          "manufacturer": "德马吉",
          "specifications": {
            "power": "15kW",
            "weight": "8000kg",
            "working_area": "800×600×500mm"
          },
          "status": "ACTIVE",
          "department": "机加工车间",
          "location": "A-01-02",
          "purchase_date": "2023-05-10",
          "warranty_period": "2025-05-10",
          "attributes": {
            "max_spindle_speed": "12000rpm",
            "tool_magazine_capacity": "30"
          }
        }
      ]
    },
    "timestamp": "2026-04-02T10:00:00Z"
  }
  ```

- **错误码**：
  
  | 错误码 | 说明 | 处理建议 |
  |--------|------|---------|
  | 400 | 请求参数错误 | 检查参数格式和取值范围 |
  | 401 | 鉴权失败 | 检查Token是否有效、数据权限是否足够 |
  | 403 | 权限不足 | 确认用户是否有访问该业务系统数据的权限 |
  | 404 | 设备不存在 | 检查设备编码是否正确，或数据尚未同步到平台 |
  | 409 | 数据版本冲突 | 平台数据与源系统不一致，需等待同步或手动刷新 |
  | 429 | 请求频率超限 | 降低请求频率，或联系平台管理员调整限流 |
  | 500 | 服务器内部错误 | 联系基础数据平台管理员 |
  | 503 | 服务暂时不可用 | 等待重试，或联系平台管理员 |

- **幂等键**：`equipment_codes` + `category` + `status` + `page` 的组合哈希
- **超时/重试/降级**：
  - 超时时间：30秒
  - 重试策略：指数退避，最多3次（1s, 2s, 4s）
  - 降级方案：从本地缓存获取数据，标记为"缓存数据"；若平台不可用，可切换直连模式（需配置）
- **版本策略**：URL版本控制（/api/v1/），向后兼容至少2个主要版本

- **错误码**：
  
  | 错误码 | 说明 | 处理建议 |
  |--------|------|---------|
  | 400 | 请求参数错误 | 检查必填字段、数据类型、取值范围 |
  | 401 | 未授权 | 用户未登录或Token过期 |

  | 409 | 实例编码重复 | 修改instance_code，保证项目内唯一 |
  | 422 | 业务规则验证失败 | 检查位置是否超出边界、属性是否符合规则 |
  | 429 | 请求频率超限 | 降低请求频率 |
  | 500 | 服务器内部错误 | 联系系统管理员 |

- **幂等键**：`project_id` + `model_id` + `instance_code` 的组合
- **超时/重试/降级**：
  - 超时时间：30秒
  - 重试策略：线性重试，最多3次（5s间隔）
  - 降级方案：异步创建，返回任务ID，通过任务查询结果
- **版本策略**：URL版本控制，请求体版本通过Content-Type指定

## D4. 一致性与补偿机制

### 一致性类型

| 场景 | 一致性类型 | 说明 | 适用接口 |
|------|-----------|------|---------|
| 布局数据保存 | 强一致性 | 保存操作必须立即生效，所有用户看到相同状态 | 项目保存、实例创建/更新/删除 |
| 业务数据同步 | 最终一致性 | 外部数据同步允许短暂延迟，但最终一致 | ERP/MES/WMS数据同步 |
| 模型文件导入 | 最终一致性 | 大文件处理需要时间，处理完成后一致 | 模型上传、版本更新 |
| 批量操作 | 最终一致性 | 批量操作部分成功时，需要补偿处理 | 批量导入、批量绑定 |

### 重试策略

| 失败类型 | 重试策略 | 最大重试次数 | 重试间隔 | 补偿措施 |
|----------|---------|------------|---------|---------|
| 网络超时 | 指数退避 | 3次 | 1s, 2s, 4s | 记录失败，提供手动重试入口 |
| 服务不可用 | 指数退避 | 5次 | 5s, 10s, 20s, 40s, 80s | 告警通知管理员，队列等待 |
| 业务错误 | 不重试 | 0次 | - | 立即失败，返回错误详情 |
| 并发冲突 | 乐观重试 | 3次 | 立即重试 | 获取最新版本后重试 |
| 数据校验失败 | 不重试 | 0次 | - | 返回验证错误，用户修正后重试 |

### 对账修复

| 对账类型 | 对账周期 | 对账方式 | 差异处理 | 负责人 |
|----------|---------|---------|---------|--------|
| 设备数量对账 | 每日 02:00 | 系统自动 | 1. 记录差异<br>2. 尝试自动修复<br>3. 生成差异报告 | 系统自动 |
| 数据绑定状态对账 | 每小时 | 系统自动 | 1. 检查绑定状态一致性<br>2. 重新绑定失败项<br>3. 告警人工干预 | 系统自动 |
| 布局完整性对账 | 每周一 09:00 | 人工抽检 | 1. 抽样检查关键设备<br>2. 手动修复差异<br>3. 更新对账规则 | IE工程师 |
| 基础数据平台数据对账 | 每日 03:00 | 系统自动 | 1. 比较平台数据与源系统关键字段<br>2. 标记不一致数据<br>3. 提供修复建议，触发平台同步 | 系统自动 |

### 死信/补偿任务（如有）

| 任务类型 | 死信条件 | 补偿机制 | 监控告警 | 保留期限 |
|----------|---------|---------|---------|---------|
| 数据同步任务 | 连续失败3次 | 1. 记录死信队列<br>2. 定时重试（1小时后）<br>3. 人工干预入口 | 邮件/钉钉告警 | 30天 |
| 模型导入任务 | 处理超时（>30分钟） | 1. 终止任务<br>2. 清理临时文件<br>3. 记录失败原因 | 系统内告警 | 7天 |
| 批量导出任务 | 文件生成失败 | 1. 清理临时文件<br>2. 记录错误日志<br>3. 通知用户重试 | 站内信通知 | 3天 |
| 外部接口调用 | 服务不可用（>1小时） | 1. 切换备用接口<br>2. 使用缓存数据<br>3. 标记"数据可能过时" | 监控大盘告警 | 永久 |
